# Rebrand to Happiness Fashion World with Nigerian visual identity

A comprehensive rebrand that renames the site from "Happiness Fashion" to "Happiness Fashion World," redesigns the logo from an HF to an HFW monogram, swaps all model imagery for Nigerian/African models via Pexels, adds three new catalog designs, and introduces two new page sections (Craftsmanship timeline and BookingCTA). Visual polish touches almost every component — gold borders, fashion-tool icons, frosted-glass stats cards, shimmer effects.

The change is entirely cosmetic and content-driven: no new dependencies, no backend logic, no auth changes. The risk profile is low, but there are a few items worth watching.

**Watch for:** Unused CSS utilities added but never consumed in components (confirmed). Hard-coded statistics in two places that can drift out of sync (confirmed). Pexels images referenced by ID without a local fallback, meaning a CDN outage or content removal breaks the entire visual layer (confirmed). Missing `@keyframes shimmer` means `.text-shimmer` will silently fail if applied (confirmed).

## High-level view

The brand rename is mechanical find-and-replace across 18+ touchpoints — title, meta tags, manifest, schema.org, footer, chatbot prompts, WhatsApp message templates. One WhatsApp message in `chatbot.ts` addresses "Happiness Fashion World!" with an exclamation mark while all others use a period, which is trivially inconsistent.

The HFW monogram is constructed as an SVG using rotated `<rect>` elements for the W strokes. The rotation transforms may render the W differently across browsers at sub-20px sizes (favicon use case) — worth testing on real devices.

Three new catalog entries expand the catalog to 12. The `featured` flag is used on 5 of 12 designs, which dilutes the visual signal when nearly half the catalog carries a gold border and "Featured" badge.

The BookingCTA repeats "200+ Happy Clients" and "7+ Years" statistics already present in the Hero stats card — hard-coded independently in two components with no shared constant. Its "View Our Process" link points to `#about` instead of `#process` (the Craftsmanship section's actual `id`).

New CSS utilities (`.text-shimmer`, `.section-divider`, `.card-glow`, `glow-pulse` keyframe) are defined but never referenced in any component. Additionally, `.text-shimmer` references a `@keyframes shimmer` that doesn't exist in the codebase — a broken animation waiting to surface.

## Unused CSS utilities and missing keyframe

Three new utilities are added to `index.css` at the `@layer components` level:

```css
.text-shimmer { /* ... */ animation: shimmer 2.5s linear infinite; }
.section-divider { /* ... */ }
.card-glow { /* ... */ }
@keyframes glow-pulse { /* ... */ }
```

None of these classes appear in any `.tsx` file in the codebase. `.text-shimmer` is the most concerning: it references `animation: shimmer 2.5s linear infinite` but the only keyframe defined is `glow-pulse`. The `@keyframes shimmer` declaration is entirely absent. If `.text-shimmer` is ever applied to an element, the gradient background will render statically with no animation — a silent failure that looks like a bug rather than missing CSS.

These should either be wired into the components that motivated them (with a proper `@keyframes shimmer` added) or removed to avoid dead code accumulation.

## Duplicated statistics between Hero and BookingCTA

The Hero's frosted-glass stats card renders:

```tsx
// Hero.tsx
<div className="font-display ...">200+</div>
<div>Clients dressed</div>
// ...
<div className="font-display ...">7 yrs</div>
<div>Of craftsmanship</div>
```

BookingCTA independently renders:

```tsx
// BookingCTA.tsx
<span>200+ Happy Clients</span>
<span>7+ Years</span>
```

These are the same business metrics stated differently ("Clients dressed" vs "Happy Clients", "7 yrs" vs "7+ Years"). When the business hits 300 clients or 8 years, two files must be updated in lockstep. A shared `STATS` constant would eliminate this coupling.

## BookingCTA "View Our Process" misdirected link

The BookingCTA's secondary button reads "View Our Process" but links to `#about`:

```tsx
<a href="#about" className="...">
  View Our Process
  <ArrowRight size={14} />
</a>
```

The Craftsmanship component — which *is* the process section — uses `id="process"`. The link should point to `#process` to match user intent.

## Pexels CDN dependency with no fallback

Every design image is constructed as:

```ts
image: pexels(8386668)  // => https://images.pexels.com/photos/8386668/...
```

The hero image, OG image in `index.html`, and schema.org image all use the same pattern. Pexels occasionally removes photos when photographers opt out, and the CDN can be slow in parts of Africa. There's no `onerror` fallback on `<img>` tags and no placeholder image defined. For a fashion portfolio where visuals are the product, this is a reliability concern worth addressing with either self-hosted copies or a fallback handler.

## Featured flag dilution

Five of twelve designs are marked `featured: true`. The `DesignCard` renders a gold top border and a "Featured" badge for these. When 42% of the catalog is featured, the grid becomes mostly gold-bordered cards, and the badge stops communicating special status. Narrowing to 2-3 featured items at a time would restore the intended emphasis.
